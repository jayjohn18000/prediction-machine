## Track A — Sport-bucketed proposer script

### A1: Identify which sports currently exist in the DB with markets on BOTH sides
Run this SQL via node/pg (use DATABASE_URL from .env) and return the results:

```sql
SELECT
  COALESCE(pm.metadata->>'sport', 'unknown') AS sport,
  p.code AS provider,
  COUNT(*) AS live_markets
FROM pmci.provider_markets pm
JOIN pmci.providers p ON p.id = pm.provider_id
WHERE pm.category = 'sports'
  AND pm.status IN ('open', 'active')
  AND (pm.close_time IS NULL OR pm.close_time > NOW())
GROUP BY 1, 2
ORDER BY live_markets DESC;
```

This tells us which sports have markets on both Kalshi AND Polymarket — those are the
only sports worth running the proposer on.

### A2: Create script `scripts/review/pmci-propose-sports-by-sport.mjs`

This script should:
1. Query the DB for distinct sport values that have markets on BOTH Kalshi and Polymarket
   (using the SQL from A1 as a subquery)
2. For each sport (e.g. 'nfl', 'nba', 'mlb', 'nhl', 'soccer', etc.) in turn:
   a. Load Kalshi markets for that sport only: `WHERE category='sports' AND metadata->>'sport'=$1`
   b. Load Polymarket markets for that sport only: same filter
   c. If either side has 0 markets, skip with a log line
   d. If either side has > 2000 markets, cap at 2000 (pick by most recent last_seen_at)
   e. Call `runProposalEngine({ category: 'sports', marketFilter: sport, dryRun: false })`
      OR if the engine doesn't support per-sport filtering, pass the loaded market lists
      directly. Check the engine's API first.
3. After each sport, log: `[sport=nfl] considered=X inserted=Y rejected=Z`
4. After all sports, log totals and run `npm run pmci:smoke` to show current_links

The engine already accepts `opts.category`. Check if it accepts pre-filtered market lists
or if it queries the DB itself. If it queries DB itself, add an `opts.sport` parameter
that appends `AND metadata->>'sport'=$sport` to the Kalshi and Polymarket queries inside
